## 슬라이드 5: GAS 소개

구글 앱스 스크립트, **GAS**가 정확히 뭘까요?

한 문장으로 정의하면, **구글 워크스페이스를 자동화하고 확장할 수 있는 클라우드 기반 JavaScript 플랫폼**입니다.

뭔가 어려워 보이죠? 쉽게 말하면, **구글 시트, 문서, Gmail, 캘린더 등을 자동으로 움직이게 만드는 도구**예요.

가장 큰 특징 세 가지를 보면요.

첫째, **무료**입니다! 구글 계정만 있으면 누구나 사용할 수 있어요.

둘째, **서버 관리가 필요 없습니다**. 구글이 다 알아서 해줘요. 여러분은 코드만 작성하면 됩니다.

셋째, **구글 서비스와의 완벽한 통합**이 가능합니다. 시트에서 데이터 읽고, Gmail로 메일 보내고, 캘린더에 일정 추가하고, 이게 다 한 번에 가능해요.

## 슬라이드 6: GAS 작동 원리

GAS가 어떻게 작동하는지 간단히 살펴볼게요.

먼저 **코드 작성** 단계입니다. 구글 시트 메뉴에서 "확장 프로그램 > Apps Script"를 클릭하면 코드 에디터가 나타납니다. 여기서 JavaScript로 코드를 작성해요.

다음은 **트리거 설정**입니다. "언제 실행할 건가요?"를 정하는 거죠. 시트를 열 때, 폼이 제출될 때, 매일 아침 9시에, 이렇게요.

그 다음 **권한 승인**을 해야 합니다. 구글이 "이 스크립트가 여러분의 시트에 접근해도 되나요?" 물어보면 "예"라고 답하는 거예요.

마지막으로 **자동 실행**입니다. 설정한 조건이 충족되면 구글의 서버에서 알아서 코드를 실행합니다. 여러분이 컴퓨터를 끄고 있어도요!

## 슬라이드 7: 구글 생태계와의 연동

GAS의 진짜 강점은 **구글 생태계와의 완벽한 연동**입니다.

**Google Sheets** - 데이터를 읽고, 쓰고, 수정하고, 차트까지 만들 수 있어요.

**Gmail** - 자동으로 이메일을 보내거나, 받은 메일을 분석하거나, 첨부파일을 다운로드할 수 있습니다.

**Google Calendar** - 일정을 생성하고, 수정하고, 알림을 보낼 수 있어요.

**Google Drive** - 파일을 생성, 복사, 이동, 공유할 수 있습니다.

**Google Forms** - 폼 응답을 자동으로 처리하거나, 응답 결과를 가공할 수 있어요.

**Google Docs** - 문서를 자동으로 생성하거나, 템플릿에 데이터를 채워 넣을 수 있습니다.

이 모든 게 **코드 몇 줄**로 가능합니다!

## 슬라이드 9: 생성형 AI 활용

이제 가장 중요한 부분입니다. **생성형 AI를 활용한 GAS 개발**!

ChatGPT나 Claude 같은 AI에게 자연어로 요청하면, AI가 코드를 작성해줍니다.

예를 들어볼게요.

"구글 시트 A열의 데이터를 B열로 복사하되, 빈 칸은 제외하고 복사하는 코드 작성해줘"

이렇게 물어보면, AI가 완성된 코드를 줍니다!

**비개발자도 AI와 대화하듯이 코드를 받을 수 있다**는 게 핵심입니다.

물론, AI가 준 코드를 **그대로 복붙**하면 안 되고요. **테스트하고, 이해하고, 수정**하는 과정이 필요합니다. 이 부분은 나중에 "AI에게 완벽한 코드 받아내는 법"에서 자세히 다룰게요.

## 슬라이드 10: GAS 핵심 기능

GAS의 핵심 기능 6가지를 살펴보겠습니다.

**1. 코드 에디터** - .gs 파일에 JavaScript 코드를 작성하고, .html 파일로 웹 UI를 만들 수 있어요.

**2. 실행 로그** - 코드가 어떻게 실행됐는지 기록을 볼 수 있습니다. 에러가 나면 여기서 확인해요.

**3. 배포** - 웹 앱으로 배포하거나, API로 만들거나, 애드온(확장 프로그램)으로 배포할 수 있습니다.

**4. 트리거** - 시간 기반(매일 9시), 이벤트 기반(폼 제출 시)으로 자동 실행을 설정할 수 있어요.

**5. 라이브러리** - 다른 사람이 만든 코드를 가져다 쓸 수 있고, 내 코드를 다른 프로젝트에서 재사용할 수 있습니다.

**6. 서비스** - 고급 구글 API(Gmail, Calendar, Drive 등)를 활성화해서 더 많은 기능을 쓸 수 있어요.

## 슬라이드 11: 시작하기

이제 실제로 어떻게 시작하는지 보여드릴게요.

**1단계: 구글 시트 열기** - 새 시트를 만들거나 기존 시트를 엽니다.

**2단계: Apps Script 접속** - 메뉴에서 "확장 프로그램 > Apps Script"를 클릭해요.

**3단계: 코드 작성** - 에디터가 열리면 function myFunction() 같은 기본 코드가 보입니다.

**4단계: 실행 버튼 클릭** - 상단의 ▶ 버튼을 누르면 코드가 실행됩니다.

**5단계: 권한 승인** - 처음 실행할 때는 권한 승인이 필요해요. (상황에 따라) "고급 > 안전하지 않은 페이지로 이동" 화면이 나타날 수 있으며, 안내에 따라 승인하면 됩니다.

진짜 **5분이면 첫 스크립트를 실행**할 수 있어요!

## 슬라이드 12: IMPORTRANGE

첫 번째 함수는 **IMPORTRANGE**입니다.

**다른 시트에서 데이터를 가져올 때** 사용해요.

문법은 간단합니다:
```
=IMPORTRANGE("스프레드시트URL", "시트명!범위")
```

예시 1: 전체 데이터 가져오기
```
=IMPORTRANGE("https://...", "매출!A1:D100")
```

예시 2: 특정 열만 가져오기
```
=IMPORTRANGE("https://...", "직원!A:C")
```

**첫 실행 시 "액세스 허용" 클릭이 필요**하고, 원본 시트 권한이 있어야 하며, 자동 갱신됩니다(지연 가능)!

## 슬라이드 13: ARRAYFORMULA

두 번째는 **ARRAYFORMULA**입니다.

**수식을 한 칸에만 입력하면 자동으로 모든 행에 적용**되는 마법 같은 함수예요.

Before (일반 방식):
```
C1에 =A1&" "&B1 입력
아래로 드래그 복사...
```

After (ARRAYFORMULA):
```
C1에 =ARRAYFORMULA(A1:A100&" "&B1:B100) 입력
끝!
```

**고급 활용**: ARRAYFORMULA + IFERROR + LEN 조합
```
=ARRAYFORMULA(IFERROR(LEN(A2:A100), 0))
```

각 셀의 글자 수를 계산하되, 에러는 0으로 표시합니다. 여러 함수를 조합하면 복잡한 작업도 한 줄로 처리 가능해요!

## 슬라이드 14: IFERROR

세 번째는 **IFERROR**입니다.

**#N/A, #DIV/0! 같은 에러 메시지 대신 원하는 값을 표시**할 때 써요.

문법:
```
=IFERROR(시도할_수식, 에러시_대체값)
```

예시 1: 0으로 나누기 방지
```
=IFERROR(A1/B1, 0)
```
B1이 0이면 에러 대신 0을 반환합니다.

예시 3: 공백으로 처리
```
=IFERROR(A1*B1, "")
```
에러 시 빈 칸으로 처리해서 시트를 깔끔하게 유지할 수 있어요.

---

## 슬라이드 15: VLOOKUP

네 번째는 **VLOOKUP**입니다.

**다른 테이블에서 값을 찾아서 가져올 때 필수** 함수예요!

문법:
```
=VLOOKUP(찾을값, 찾을범위, 가져올열번호, FALSE)
```

예시: 직원명으로 부서 찾기
```
=VLOOKUP(A1, Sheet2!A:C, 2, FALSE)
```

**고급 활용**: ARRAYFORMULA + IFERROR + LEN 조합
```
=ARRAYFORMULA(
  IF(LEN(A2:A)>0,
    IFERROR(
      VLOOKUP(A2:A, Sheet2!A:B, 2, FALSE),
      "미등록"
    ),
    ""
  )
)
```

빈 셀이 아니면 조회하고, 없으면 "미등록", 빈 셀은 빈칸으로 처리합니다!

## 슬라이드 16: 기타 유용한 함수

마지막으로 자주 쓰는 함수 2개만 더 보고 갈게요.

**1. FILTER - 조건 필터링**
```
=FILTER(A2:C100, C2:C100>=1000000)
```
매출 100만원 이상만 자동으로 추출합니다.

**3. UNIQUE - 중복 제거**
```
=UNIQUE(A2:A100)
```
리스트에서 중복값을 자동으로 제거합니다.

이 정도만 알아도 **GAS 시작하기에 충분**합니다!

## 슬라이드 17: 코드 편집기 인터페이스

Apps Script 에디터에 처음 들어가면 어떤 화면이 보이는지 살펴볼게요.

**왼쪽 패널**에는 파일 목록(Code.gs, index.html), 트리거, 프로젝트 설정이 있습니다.

**중앙 편집 영역**에서 코드를 작성하는데, 문법 하이라이팅과 자동완성을 지원해요.

**상단 버튼**은 중요합니다!
- ▶ 실행: 선택한 함수를 실행
- 🐛 디버그: 디버깅 모드로 천천히 실행
- 💾 저장: Ctrl+S
- 🚀 배포: 웹앱이나 API로 배포할 때 사용

**파일 타입**은 두 가지예요.
- .gs: JavaScript 코드
- .html: 웹 UI

유용한 단축키도 알려드리면요, Ctrl+클릭하면 함수 정의로 이동하고, Ctrl+/는 주석 토글, Ctrl+F는 찾기입니다!

## 슬라이드 18: 라이브러리 & 서비스

**라이브러리**는 다른 사람이 만든 코드를 재사용하는 기능입니다.

사용 방법은 왼쪽 패널에서 "라이브러리 +" 클릭하고, 스크립트 ID를 입력한 다음, 버전을 선택하면 돼요.

**서비스**는 고급 구글 API를 사용하는 기능입니다.

주요 서비스로는 Gmail API, Calendar API, Drive API, Sheets API가 있어요.

추가 방법은 왼쪽 패널에서 "서비스 +" 클릭하고, 원하는 서비스를 검색해서 추가하면 됩니다.

이 두 가지를 활용하면 **코드를 처음부터 작성하지 않고 효율적으로 개발**할 수 있어요!

## 슬라이드 19: 트리거

**트리거**는 스크립트를 자동으로 실행하는 설정입니다.

**시간 기반 트리거**는 특정 시간에 자동 실행되는 거예요. 매일 오전 9시, 매주 월요일, 매월 1일 같은 식으로요.

**이벤트 기반 트리거**는 특정 이벤트가 발생할 때 실행됩니다.
- onOpen: 시트를 열 때
- onEdit: 시트를 수정할 때
- onFormSubmit: 폼이 제출될 때

설정 방법은 왼쪽 패널에서 ⏰ 트리거를 클릭하고, "트리거 추가"를 선택한 다음, 함수와 이벤트를 선택하면 돼요.

자주 발생하는 오류 세 가지:
1. **함수명 오타**: 트리거에 지정한 함수명과 실제 함수명이 달라요
2. **권한 미승인**: 트리거 실행 전에 권한 승인을 안 했어요
3. **할당량 초과**: (트리거/스크립트 총 실행 시간 기준) 일반 계정 90분/일, Workspace 계정 6시간/일

## 슬라이드 20: 에러가 생겼을 때 어떻게 해결하나요?

자, 이제 중요한 내용입니다. **에러가 생겼을 때 어떻게 해결하나요?**

핵심 메시지는 이겁니다: **코드가 안 돌아가는 건 당연합니다!** 개발자도 매일 에러를 만나요. 차이는 "에러 메시지를 읽고 AI에게 물어본다"는 것!

**로그(Log)가 뭔가요?** 
비행기 블랙박스처럼, 코드가 실행되는 과정을 기록하는 것입니다.

**디버깅(Debugging)이 뭔가요?**
벌레(Bug)를 잡는다는 뜻! 코드의 문제를 찾아서 고치는 것이에요.

비개발자를 위한 해결 프로세스:
1. 에러 발생!
2. 로그 확인 (무슨 에러인지 읽기)
3. AI에게 물어보기 (ChatGPT/Claude에게 복붙)
4. 해결! (AI가 알려준 대로 수정)

코드를 직접 고칠 필요 없어요! **에러 메시지만 잘 읽고 AI에게 전달하면 끝**입니다!

## 슬라이드 21: 에러 로그 확인하고 AI에게 알려주기

구체적으로 어떻게 하는지 볼게요.

**에러 로그는 어디서 볼까요?**

방법 1: 실행 로그 패널
1. 왼쪽 패널에서 "실행 로그" 클릭
2. 최근 실행 기록에서 빨간색 ❌ 표시 찾기
3. 클릭하면 에러 메시지 나옴

방법 2: 실행 후 바로
1. 코드 실행 (▶ 버튼 클릭)
2. 하단에 나타나는 팝업 확인
3. "로그 보기" 또는 "실행 로그" 클릭

**AI 프롬프트 예시**는 이렇게 작성하면 돼요:

```
"구글 앱스 스크립트로 시트의 A열 데이터를 읽으려고 했는데 에러가 났어.

에러 메시지:
TypeError: Cannot read properties of null (reading 'getRange')
    at myFunction(Code.gs:5)

문제가 발생한 코드:
const sheet = SpreadsheetApp.getActiveSheet();
const data = sheet.getRange('A1:A10').getValues();

뭐가 문제일까? 어떻게 고쳐야 해?"
```

추가 팁:
- 스크린샷도 함께 보내면 AI가 더 정확하게 이해해요
- 에러가 반복되면 전체 코드도 함께 공유하세요
- AI가 준 해결책을 그대로 적용하고, 안 되면 다시 물어보세요!

## 슬라이드 22: 스크립트 속성

**스크립트 속성**은 API 키, 비밀번호, URL 등을 안전하게 저장하는 공간입니다.

왜 필요하냐면요.

나쁜 방법은 코드에 API 키를 직접 작성하는 건데, 이러면 공유 시 유출 위험이 있어요.

좋은 방법은 스크립트 속성에 저장하는 거예요. 코드를 공유해도 안전합니다!

설정 방법:
1. 왼쪽 패널 "프로젝트 설정" (⚙️) 클릭
2. "스크립트 속성" 탭 선택
3. "속성 추가" 클릭
4. 속성(키)과 값 입력

속성 유형 3가지:
- **스크립트 속성**: 모든 사용자 공통
- **사용자 속성**: 사용자별 개별 저장
- **문서 속성**: 특정 문서에만 저장

> 참고: UI(프로젝트 설정 화면)에서 직접 추가/수정이 가능한 것은 **스크립트 속성**뿐입니다. **사용자/문서 속성**은 보통 코드에서 `PropertiesService`로 관리합니다.

## 슬라이드 23: 배포

마지막으로 **배포** 방법입니다.

내 스크립트를 세상에 공개하는 방법이에요!

**1. 웹 앱으로 배포**
- 용도: URL로 접근 가능한 웹사이트
- 필수: doGet() 또는 doPost() 함수
- 예시: 설문조사 폼, 대시보드, 데이터 조회 페이지

**2. API 실행으로 배포**
- 용도: 외부에서 함수를 호출할 수 있는 API
- 인증: **OAuth 2.0** 기반 사용자/서비스 계정 인증 필요 (API 키는 필수 아님)
- 예시: 데이터 처리 API, 자동화 트리거

**3. 애드온(Add-on)으로 배포**
- 용도: Google Workspace Marketplace에 게시
- 필수: 승인 절차
- 예시: 시트/문서 확장 기능

대부분은 **1번 웹 앱 배포**를 가장 많이 사용합니다!

## 슬라이드 24: 비상연락망 프로필 카드

첫 번째 사례는 **비상연락망 프로필 카드 자동 생성**입니다.

**문제 상황**: 비상연락망을 웹페이지로 만들고 싶은데, 일일이 HTML 코딩하기 번거로워요.

**해결 방법**: 구글 시트에 이름, 전화번호, 부서 입력 → GAS가 자동으로 예쁜 웹페이지 생성!

3. 웹 앱으로 배포하면 URL로 접근 가능

**핵심 아이디어**는 "데이터와 디자인을 분리"하는 겁니다. 데이터는 시트에서 관리하고, 디자인은 HTML 템플릿으로 만들어요.

**이런 점이 좋아요**:
- 비개발자도 시트만 수정하면 자동 업데이트

## 슬라이드 25: 비상연락망 사진 링크

두 번째 사례는 **구글 드라이브 사진 링크 자동 생성**입니다.

**문제 상황**: 직원 사진을 웹페이지에 보여주고 싶은데, 파일 이름이 제각각이고 링크 복사하기 번거로워요.

**해결 방법**: 드라이브에 사진 업로드 → GAS가 파일명 정리하고 공유 링크 자동 생성 → 시트에 저장!

**어떻게 작동하나요?**

3. 각 파일의 공유 링크를 생성
4. 시트에 [이름, 링크] 형태로 저장

**핵심 아이디어**는 "일관된 파일명 = 쉬운 관리"입니다.

**이런 점이 좋아요**:
- 수백 개 파일도 1분 안에 처리
- 사진 추가 시 버튼 한 번으로 업데이트

## 슬라이드 26: 시트 데이터 재가공

**해결 방법**: 원본 시트 → GAS로 필터링/정렬/계산 → 새 시트에 저장!

**어떻게 작동하나요?**

4. 계산 (예: 합계, 평균)
5. 가공된 결과를 다른 시트에 저장

**핵심 아이디어**는 "원본 보호 + 유연한 가공"입니다.

**이런 점이 좋아요**:
- 원본 데이터 훼손 방지
- 복잡한 조건도 코드로 자유롭게 처리
- 정기적으로 자동 실행 가능

## 슬라이드 32: AI에게 GAS 코드 완벽하게 받아내는 법

자, 이제 가장 중요한 부분입니다!

**AI에게 GAS 코드를 완벽하게 받아내는 4단계 프로세스**를 알려드릴게요.

**왜 중요한가?**
AI가 코드를 작성해줘도, **명확한 요청 = 정확한 코드 = 시간 절약**이거든요.

**4단계 프로세스**:

1️⃣ **사전 준비** - 데이터 구조 파악, 목표 명확화, 레퍼런스 찾기

2️⃣ **맥락 제공** - 스크린샷 활용, 파일 공유, 상황 설명

3️⃣ **구체적 요청** - 데이터 흐름 명확히, 원하는 결과 명시, 제약사항 공유, 작동 순서 상세히

4️⃣ **꼼꼼히 검증** - 데이터 검증(개수, 값, 형식), 기능 검증(정상/예외 케이스), 성능 검증(속도, 할당량)

**프롬프트 템플릿**:
```
"[상황] 구글시트에 [어떤 데이터]가 있습니다.
[원하는 동작] [트리거]이 발생하면, [어떤 데이터]를 [어떻게 처리]해서 [어디로] 출력하고 싶습니다.
[제약] 구글앱스스크립트만 사용, 외부 서버 불가"
```

## 슬라이드 33: 추가 실전 팁

더 나은 코드를 받는 법을 알려드릴게요.

**DO's (이렇게 하세요)**:
- 데이터 구조를 명확히 파악하고 공유
- 스크린샷과 예시를 충분히 제공
- 제약 조건을 미리 알려주기
- 단계별로 나눠서 요청
- 테스트 데이터로 먼저 검증
- 주석 많이 달아달라고 요청
- 작동하는 코드는 백업
- 에러 로그 전체 복사해서 공유

**DON'Ts (하지 마세요)**:
- "알아서 해줘" 같은 막연한 요청
- 한 번에 모든 기능 요청
- 실제 데이터로 바로 테스트
- 에러나도 로그 안 보고 "안 돼요"만 말하기
- AI가 준 코드 이해 안 하고 무조건 복붙
- 백업 없이 기존 코드 덮어쓰기
- 검증 없이 운영 환경에 바로 적용

## 슬라이드 35: Q&A

자주 묻는 질문들을 미리 정리해봤어요.

**Q1. GAS는 무료인가요?**
A. 네, 완전 무료입니다! 단, 실행 시간은 하루 90분(일반 계정) / 6시간(Workspace 계정)으로 제한됩니다.

**Q2. JavaScript를 몰라도 되나요?**
A. AI를 활용하면 JavaScript를 몰라도 충분히 만들 수 있습니다. 단, 기본 개념은 이해하는 게 좋아요.

**Q3. 만든 스크립트를 다른 사람과 공유할 수 있나요?**
A. 네! 스크립트를 복사해서 공유하거나, 라이브러리로 배포하거나, 웹 앱으로 만들어 URL 공유가 가능합니다.

**Q4. 실시간 자동화가 가능한가요?**
A. 시간 기반 트리거는 가능하지만, 초 단위 실시간은 어렵습니다. 가장 짧은 간격이 1분입니다.

**Q5. 슬랙, Notion 등과 연동이 쉬운가요?**
A. UrlFetchApp.fetch()로 API 호출이 가능합니다. AI에게 물어보면 쉽게 코드를 받을 수 있어요!
